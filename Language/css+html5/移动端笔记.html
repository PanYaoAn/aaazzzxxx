	移动端的点透问题
        当上层元素发声点击的时候,下层元素也有点击(焦点)特性,在300毫秒之后如果上层元素消失或隐藏,目标点就会''漂移到下层元素
        身上,就会触发点击行为
		解决:
        1.不要使用有点击(焦点)特性的元素
        2.阻止pc事件
		3.使用移动端事件

##阻止pc事件 禁止PC事件代码 document.addEventListener('touchstart',function(ev){ ev.preventDefault(); }); 1.IOS10下设置meta禁止用户缩放是不可行可以使用阻止pc事件就可以在IOS10下禁止用户缩放 2.解决IOS10下溢出隐藏的问题 3.禁止系统默认的滚动条.橡皮筋效果 4.可以禁止长按选中文字,选中图片 5.解决点透问题 6.也阻止了焦点元素的焦点行为 需要焦点的元素.addEventListener('touchstart',function(ev){ ev.stopPropagation(); })

##TouchEvent 事件对象 #####1.touches 当前位于屏幕上的所有手指的一个列表 div.addEventListener('touchmove',start); function start(ev){ this.innerHTML = ev.touches.length; }

#####2.targetTouches 位于DOM元素上的手指一个列表 div.addEventListener('touchmove',start); function start(ev){ this.innerHTML = ev.targetTouches.length; }

#####3.changedTouches 涉及当前事件的手指一个列表 div.addEventListener('touchmove',start); function start(ev){ this.innerHTML = ev.changedTouches.length; }
